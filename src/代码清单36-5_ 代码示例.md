# 多智能体客服系统 角色定义文档

## 1. 系统概述

**系统名称：** 企业智能客服系统
**核心功能：** 处理客户咨询、订单查询、物流跟踪、投诉建议
**目标用户：** 企业客户、内部员工

## 2. 角色定义

### 2.1 总控Agent（Router）

**表36-3　Router角色定义**

| 属性 | 定义 |
|-----|------|
| 角色名称 | 总控Agent |
| 角色ID | router |
| 职责范围 | 意图识别、任务分流、结果聚合 |
| 输入格式 | 用户原始文本 |
| 输出格式 | 结构化任务请求 |
| 可用工具 | 意图分类器、任务分发器 |
| 限制规则 | 不直接回答具体业务问题 |

### 2.2 订单Agent（Order）

**表36-4　Order角色定义**

| 属性 | 定义 |
|-----|------|
| 角色名称 | 订单专员 |
| 角色ID | order_agent |
| 职责范围 | 订单查询、状态更新、订单历史 |
| 输入格式 | `{order_id, user_id, query_type}` |
| 输出格式 | `{order_status, items, amount, created_at}` |
| 可用工具 | 订单数据库查询API |
| 限制规则 | 不能取消订单、不能修改金额 |

### 2.3 物流Agent（Logistics）

**表36-5　Logistics角色定义**

| 属性 | 定义 |
|-----|------|
| 角色名称 | 物流专员 |
| 角色ID | logistics_agent |
| 职责范围 | 物流跟踪、时效分析、最优方案推荐 |
| 输入格式 | `{order_id, destination, urgency}` |
| 输出格式 | `{shipping_options, estimated_days, costs}` |
| 可用工具 | 物流API、仓库库存API |
| 限制规则 | 不能触发实际发货 |

### 2.4 投诉Agent（Complaint）

**表36-6　Complaint角色定义**

| 属性 | 定义 |
|-----|------|
| 角色名称 | 投诉处理专员 |
| 角色ID | complaint_agent |
| 职责范围 | 投诉记录、工单创建、安抚处理 |
| 输入格式 | `{user_id, complaint_type, description}` |
| 输出格式 | `{ticket_id, status, estimated_response_time}` |
| 可用工具 | 工单系统API、用户信息API |
| 限制规则 | 不能承诺赔偿金额、不能泄露其他用户信息 |
